**Duration:** ~4 min

**Talking points**
- Bitstream Studio is not three separate apps — it is one webview bundle with three workspace modes.
- Sensor Telemetry is the operator console for BS2: connect, route, configure, inspect samples.
- Sensor Studio is the creative layer: flows, Stage, Dashboard — still fed by the same live store.
- Presentation is the training layer: theory on screen, demos read the store (no second WebSocket).
- In VS Code you can open Presentation in a **second panel** for projector / side-by-side teaching.

**Demo script**
- Switch tabs in the toolbar (or Ctrl+Shift+1/2/3 in browser dev).
- Show that bridge connection status persists across workspace switches.

**Q&A prompts**
- Why not a standalone presentation app with its own decoder?
- When would you use the separate Presentation panel vs the workspace tab?
